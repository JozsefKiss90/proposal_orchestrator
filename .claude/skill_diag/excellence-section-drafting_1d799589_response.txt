      "content": "Underlying concepts, models, and assumptions\n\nMAESTRO's methodology integrates four theoretical paradigms that are each necessary and that must be co-designed to achieve robust long-horizon agent behaviour:\n\n(1) Neuro-symbolic hybrid reasoning (Pillar 1, WP2): The planning engine (WP2) combines LLM-based reasoning — effective at interpreting high-level natural language instructions — with symbolic planning algorithms (Monte Carlo Tree Search, hierarchical task networks) that provide formal guarantees. Integration is achieved through LLM-driven heuristic evaluation within symbolic search: the LLM provides cost and feasibility estimates while the symbolic planner maintains formally verifiable plan structures with pre-conditions, post-conditions, and verification checkpoints. The core assumption is that neither paradigm alone is adequate — LLMs provide linguistic flexibility but produce plans that degrade unpredictably over long horizons; symbolic planners provide correctness but lack natural language grounding. Self-correction through plan monitoring enables replanning from verified intermediate states rather than restarting from scratch.\n\n(2) Unified adaptive memory architecture (Pillar 2, WP3): Working memory (structured short-term store maintaining current goals, active plan state, and recent observations), episodic memory (retrieval-augmented store of past task episodes indexed by context similarity, enabling experience-based agent behaviour without retraining), and semantic memory (persistent knowledge graph connecting domain concepts, tool capabilities, and learned task patterns) are integrated with a learned memory controller governing commit, retrieval, and forgetting decisions. The architecture assumes that different interaction phases require different memory access patterns and that dynamic routing to the appropriate store is necessary to balance recall accuracy against computational cost.\n\n(3) Formal multi-agent coordination theory (Pillar 3, WP4): Task-delegation protocols are specified using mechanism design and distributed algorithm theory, enabling mathematical proofs of convergence. Conflict resolution under partial observability uses game-theoretic mechanisms adapted from the multi-agent systems literature. Every delegation, negotiation step, and joint decision produces a structured trace for human inspection and accountability.\n\n(4) Principled external tool and API integration (cross-cutting, all WPs): The Orchestration Layer addresses the mandatory scope requirement SR-04 as an architectural substrate. The typed tool registry and formal selection policies make tool invocation a first-class element of plan construction integrated with the planning engine. This design assumption — that external tool integration must be architecturally central and not a bolt-on function-calling interface — drives the design of all three pillars: the planning engine decides when tools are needed; the memory architecture stores tool capabilities, execution histories, and learned tool-selection heuristics; and the coordination layer governs shared tool access across collaborating agents.\n\nWork package methodology\n\nThe WP structure directly implements this methodology. WP2 (lead: ATU, tasks T2-01 to T2-03) develops the neuro-symbolic planning engine through sub-task graph decomposition design, MCTS and HTN integration with LLM heuristics, and plan monitoring and self-correction implementation — delivering OBJ-1 and providing the planning substrate consumed by WP5, WP6, and WP7. WP3 (lead: BIIS, tasks T3-01 to T3-03) implements the tri-store memory architecture through design, indexing and retrieval implementation, and forgetting mechanism development — delivering OBJ-2 and providing the memory substrate across all demonstrators. WP4 (lead: CERIA, tasks T4-01 to T4-03) develops the coordination protocols through formal specification, convergence analysis under partial observability, and auditable trace module implementation — delivering OBJ-3 and providing the coordination layer for WP5, WP6, and WP7. WP5 (NIHS), WP6 (ISIA), and WP7 (ELI) validate the fully integrated architecture in operational healthcare, manufacturing, and logistics environments, delivering OBJ-4, OBJ-5, and OBJ-6 respectively. WP8 (lead: FIIT) provides the cross-cutting evaluation framework, integration benchmarking, and standardised benchmark suite. WP9 (lead: BAL) executes open-source release management and AI-on-demand platform contributions.\n\nKey challenges and mitigation\n\nThree principal technical challenges bear directly on the methodology. (1) Scalability of formal verification: proofs of convergence and plan correctness may become computationally intractable for large state spaces. Mitigation: bounded planning horizons and approximate verification with formally quantified error bounds developed in WP2 and WP4. (2) Memory coherence under extended interactions: irrelevant information accumulation degrades retrieval precision over long interaction sequences. Mitigation: the learned forgetting mechanism (T3-03, WP3) combined with context-similarity-based episodic indexing addresses this directly. (3) Heterogeneous API integration complexity: HL7 FHIR, OPC-UA, and logistics tracking APIs present distinct data models and reliability profiles across the three demonstrator domains. BIIS contributes from a Horizon Europe project on unified transformers for text and retrieval. CERIA builds on an H2020 autonomous multi-agent exploration project and an ERC Advanced Grant on formal methods for multi-agent systems. NIHS contributes from a Horizon Europe project on European Health Data Space pilot implementation. ISIA builds on a Horizon Europe project on AI for manufacturing process optimisation. FIIT builds on an H2020 project contributing to the AI-on-demand platform. BAL builds on a Horizon Europe project on European large language models. The project aligns with and contributes to the European Partnership on AI, Data and Robotics (ADRA) and the GenAI4EU central Hub (CSA HORIZON-CL4-2025-03-HUMAN-18) through dedicated cohesion tasks in WP1 (T1-03), as required by the call scope.\n\nInterdisciplinary approach\n\nMAESTRO integrates four disciplines that are each essential to achieving the project's objectives: (1) formal computer science and AI planning theory (HTN planning, MCTS, mechanism design, formal verification) — centred in ATU and CERIA; (2) deep learning and neural information retrieval (memory-augmented neural networks, retrieval-augmented generation, knowledge graph embedding) — centred in BIIS; (3) large language model engineering (fine-tuning, tool-augmented architectures, production-scale deployment) — centred in BAL; and (4) domain engineering in healthcare informatics (HL7 FHIR, clinical NLP, AI ethics in healthcare, diagnostic benchmarking) centred in NIHS, smart manufacturing (digital twins, OPC-UA, Industry 4.0) centred in ISIA, and logistics AI (supply chain optimisation, multi-actor coordination) centred in ELI. The integration is structural, not interfacial: the neuro-symbolic planning engine is co-developed by ATU (formal planning methods) and BAL (LLM engineering); the adaptive memory system is co-developed by BIIS (neural retrieval architectures) and ATU (formal pre/post-condition specification); multi-agent protocols are co-developed by CERIA (formal multi-agent systems) and ATU (neuro-symbolic integration). Demonstrator benchmark scenarios and validation criteria are co-defined with NIHS, ISIA, and ELI, embedding domain knowledge at the design level rather than treating it as a late-stage test condition.\n\nSocial sciences and humanities\n\nMAESTRO is primarily a technical AI research project developing agent architectures for planning, memory, and coordination. No dedicated SSH disciplinary research tracks are proposed, as the project's research gaps lie in planning theory, memory architectures, and formal coordination protocols rather than in social science phenomena.\n\nGender dimension in research and innovation content\n\nThe application form (B.1.2) mandates consideration of the gender dimension in research and innovation content.
