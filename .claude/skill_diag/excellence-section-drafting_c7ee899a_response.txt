A learned memory controller governs information flow between all three stores, determining what to commit to long-term storage, what to retrieve, and when to forget — balancing recall accuracy against computational cost. The underlying model assumes that the three-store architecture mirrors established cognitive memory models and that retrieval-augmented approaches can efficiently index agent experience at scale. Deliverables: D3-01 (memory architecture design specification, Month 12) and D3-02 (adaptive memory system software, Month 30).\n\nPillar 3 — Decentralised Multi-Agent Coordination (WP4, CERIA lead; contributing partners ATU, BIIS):\nTasks T4-01 through T4-03 develop: a formal task-delegation protocol in which agents decompose tasks, negotiate subtask assignments, and establish shared commitment structures with provable convergence under partial observability; conflict resolution mechanisms for situations where agent plans interfere, resources are contested, or partial observations lead to inconsistent world models; and an auditable multi-agent interaction trace module producing structured traces that human operators can inspect and understand.
