      "content": "MAESTRO adopts a co-design methodology in which all three research pillars — Pillar 1 (neuro-symbolic planning and reasoning, WP2), Pillar 2 (adaptive memory architecture, WP3), and Pillar 3 (decentralised multi-agent coordination, WP4) — and the cross-cutting External Tool and API Orchestration Layer are designed from project inception to share common data representations, invocation interfaces, and evaluation benchmarks. The integration is the project's central methodological commitment: the planning engine uses the memory architecture for long-horizon coherence; the memory controller stores tool capabilities and invocation histories for the orchestration layer; and the multi-agent coordination layer uses planning and memory to manage heterogeneous agent collaboration. Research follows a design–implement–verify cycle with 6-month integration checkpoints coordinated by WP8 (FIIT lead) and evaluated against the standardised MAESTRO benchmark suite across all demonstrators.\n\nPillar 1 — Neuro-Symbolic Planning and Reasoning (WP2, ATU lead): The methodology combines the flexibility of LLM-based reasoning with formal guarantees from symbolic planning through three mechanisms: (i) goal decomposition into sub-task graphs with formally specified pre-conditions, post-conditions, and verification checkpoints at each node; (ii) integration of Monte Carlo Tree Search (MCTS) and hierarchical task networks (HTN) with LLM-driven heuristic evaluation to explore solution spaces efficiently; and (iii) plan monitoring that compares runtime execution state against the formal goal specification and triggers replanning from the most recent verified intermediate state when failures are detected, avoiding full restarts and reducing cascading failures over long task horizons.\n\nPillar 2 — Adaptive Memory Architecture (WP3, BIIS lead): The tri-store memory methodology employs: working memory as a structured, attention-managed short-term store maintaining current goals, active plan state, and recent observations; episodic memory as a retrieval-augmented store of past task episodes indexed by context similarity, enabling experience-based agent behaviour without retraining; and semantic memory as a persistent knowledge graph connecting domain concepts, tool capabilities, and learned task patterns. A learned memory controller governs information flow between stores, balancing recall accuracy against computational cost through learned routing policies. The unified architecture enables agents to maintain coherent reasoning over extended interactions that exceed the context window of any single LLM call.\n\nPillar 3 — Decentralised Multi-Agent Coordination (WP4, CERIA lead): Coordination protocols are formally specified using game-theoretic mechanism design and distributed constraint optimisation. The task-delegation protocol enables agents to decompose tasks, negotiate subtask assignments, and establish shared commitment structures with convergence proved formally using tools from distributed systems theory. Conflict resolution mechanisms handle situations where agent plans interfere or partial observations lead to inconsistent world models. Every delegation, negotiation step, and joint decision produces a structured, inspectable audit trace, addressing the transparency requirements for trustworthy AI deployment in regulated sectors.\n\nExternal Tool and API Orchestration Layer (cross-cutting, WP2, ATU lead): The layer provides a typed tool registry with machine-readable capability models including input/output schemas, reliability metadata, and cost profiles; formal tool selection and invocation policies integrated with the planning engine; standardised API adapters for REST, GraphQL, HL7 FHIR (healthcare), and OPC-UA (manufacturing); post-invocation verification of tool outputs against expected schemas before incorporation into agent state; runtime monitoring with timeout detection, error classification, and automated fallback strategies; and full provenance tracking for every tool invocation, stored in the adaptive memory architecture.\n\nConnection to prior R&I activities: MAESTRO builds on outputs from prior EU-funded projects accessible to the consortium: ATU's prior Horizon Europe contributions to the AI-on-demand platform; CERIA's ERC Advanced Grant on formal methods for multi-agent systems; BIIS's Horizon Europe project on unified transformers for text and retrieval; NIHS's Horizon Europe project on the European Health Data Space pilot implementation; and FIIT's Horizon Europe project on AI testing and evaluation frameworks. The project links to AI Factories and Data Labs for compute-intensive training and large-scale experimentation consistent with SR-11.\n\nInterdisciplinary approach: The research questions require simultaneous integration of computer science, machine learning, automated planning and formal methods, information retrieval, game theory and mechanism design, clinical informatics, and industrial engineering. Each discipline contributes essential methods: neuro-symbolic planning requires both formal methods and LLM engineering; clinical decision support requires clinical informatics alongside AI architecture; and manufacturing optimisation requires industrial engineering and digital twin modelling alongside multi-agent coordination. The consortium is structured to cover each discipline: ATU (neuro-symbolic AI, automated planning), CERIA (formal methods, multi-agent systems), BIIS (neural memory architectures, retrieval-augmented generation), NIHS (clinical informatics, health data interoperability), ISIA (AI for manufacturing, digital twins), BAL (LLM engineering, open-source deployment), and FIIT (evaluation methodology, technology transfer). No single discipline is sufficient; the research questions span all of them.\n\nSocial sciences and humanities: MAESTRO is a technical AI research project whose core methodological contributions are computational. SSH disciplines do not provide foundational research methods for the neuro-symbolic planning engine, memory architecture, or coordination protocol design. The ethics, accountability, and transparency dimensions of the research are addressed through established mechanisms: WP1's ethics compliance oversight (T1-03), including coordination with the GenAI4EU Hub; NIHS's clinical research ethics review infrastructure and data governance for the healthcare demonstrator; and FIIT's responsible AI assessment expertise in WP8 evaluation. These mechanisms embed ethical practice into the research without requiring SSH disciplinary research contributions that are not substantiated by the project's scientific approach.\n\nGender dimension in research and innovation content: The clinical decision support demonstrator (WP5, NIHS lead) validates AI-assisted diagnostic reasoning on 500 de-identified clinical cases spanning five diagnostic domains. Clinical AI validation on patient cohorts requires sex and gender as variables in benchmark design, since diagnostic pathways and clinical outcomes can differ by biological sex and by socially determined gender. The sex/gender stratification of the WP5 clinical benchmark (T5-01: clinical use case definition and diagnostic benchmark design) is addressed within the ethics compliance and benchmark design process under WP1's ethics compliance oversight (T1-03) and NIHS's established clinical ethics review processes. The foundational technical research on planning, memory, and coordination architectures does not involve human subjects or sex/gender-differentiated data, and sex/gender analysis is not applicable to those core architectural outputs.\n\nOpen science practices: MAESTRO adopts open science across all output types. All software outputs — the neuro-symbolic planning engine, adaptive memory system, coordination protocol implementation, External Tool and API Orchestration Layer, and unified MAESTRO framework — are released open-source under permissive licences by month 36 (D9-02, BAL lead, WP9). All research findings are published open access in top AI venues (NeurIPS, ICML, ICLR, AAMAS, AAAI). All benchmarks and demonstrator results are contributed to the AI-on-demand platform. The MAESTRO benchmark suite (D8-02, FIIT lead) is designed for community adoption as a standardised evaluation resource. A FAIR data management plan (D1-02, ATU, WP1) is delivered by month 6, ensuring research data is findable, accessible, interoperable, and reusable in line with Horizon Europe requirements.",
