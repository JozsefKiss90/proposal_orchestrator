      "content": "Overall methodology and conceptual foundation. MAESTRO's methodology is grounded in the co-design thesis: that structured planning, persistent memory, multi-agent coordination, and principled tool integration must be designed as a unified system — sharing common data representations, verification interfaces, and integration contracts — to produce agents that are robust, trustworthy, and deployable in high-value real-world settings. This thesis is operationalised across WP2 to WP4 (research pillars) unified by a cross-cutting External Tool and API Orchestration Layer, with validation in WP5 to WP7 (sector demonstrators) and integration and benchmarking managed by WP8.\n\nPillar 1 — Neuro-symbolic planning and reasoning (WP2, ATU lead). Task T2-01 develops sub-task graph decomposition with formally specified pre-conditions, post-conditions, and verification checkpoints — enabling structured verification at each planning step. Task T2-02 integrates Monte Carlo Tree Search (MCTS) and hierarchical task networks (HTN) with LLM-driven heuristic evaluation, enabling efficient exploration of structured solution spaces while retaining the flexibility of neural reasoning. Task T2-03 implements plan monitoring with verifiable self-correction: detecting execution failures and replanning from verified intermediate states rather than restarting from scratch — directly addressing the plan fragility over long task horizons that afflicts current LLM chain-of-thought approaches. The core methodological assumption is that formal pre/post-condition specification enables verifiable plan recovery, making this approach qualitatively different from prompt-engineering-based agent planning. Key output: neuro-symbolic planning framework (D2-01 architecture specification, M18; D2-02 planning engine software, M36).\n\nPillar 2 — Adaptive memory architecture (WP3, BIIS lead). Task T3-01 designs the unified tri-store architecture: working memory (attention-managed short-term store maintaining current goals, active plan state, and recent observations, analogous to a cognitive working memory buffer); episodic memory (retrieval-augmented store of past task episodes indexed by context similarity, enabling experience-based agent behaviour without retraining); and semantic memory (persistent knowledge graph connecting domain concepts, tool capabilities, and learned task patterns, continuously updated through agent interactions). Task T3-02 implements memory indexing and retrieval strategies for efficient episode recall; T3-03 develops forgetting mechanisms and coherence maintenance protocols. A learned memory controller governs information flow between stores, balancing recall accuracy against computational cost to sustain coherent, context-aware agent behaviour over interaction sequences exceeding 50 agent steps. Key output: adaptive memory system (D3-01 design specification, M12; D3-02 software, M30).\n\nPillar 3 — Decentralised multi-agent coordination (WP4, CERIA lead). Task T4-01 defines formally specified task-delegation and negotiation protocols enabling agents to decompose tasks, negotiate subtask assignments, and establish shared commitment structures with provable convergence. Task T4-02 addresses convergence and conflict resolution under partial observability, where agent plans may interfere, resources are contested, or inconsistent world models emerge from distributed sensing. Task T4-03 implements auditable multi-agent interaction trace modules, ensuring every delegation, negotiation step, and joint decision produces a structured trace that human operators can inspect and audit. Convergence guarantees are mathematically proven rather than empirically observed, distinguishing MAESTRO from ad-hoc LLM-based agent communication patterns that lack formal accountability. Key output: coordination protocol specification (D4-01, M18) and decentralised coordination software (D4-02, M36).\n\nCross-cutting — External Tool and API Orchestration Layer. Overcoming the limitations of large AI models in tasks requiring accuracy, reliability, and domain-specific precision requires principled integration of external tools and APIs as a core architectural element (SR-04). The layer provides: a typed tool registry and capability model with machine-readable input and output schemas, preconditions, postconditions, cost profiles, and reliability metadata that agents can query, reason over, and update; formal tool selection and invocation policies integrated with the neuro-symbolic planning engine (Pillar 1), determining when and why to invoke an external tool versus relying on internal agent reasoning; standardised API adapters abstracting heterogeneous external protocols — REST and GraphQL for general APIs, HL7 FHIR for healthcare clinical data systems, and OPC-UA for manufacturing industrial systems — into a uniform invocation interface; post-invocation verification and grounding, validating tool outputs against expected schemas and domain constraints before incorporating results into agent state, grounding agent reasoning in verified external data; execution monitoring with timeout detection, error classification, and automated fallback strategies integrated with the planning engine's self-correction capability; and auditable tool execution traces stored in episodic memory (Pillar 2) for experience-based tool selection and in the multi-agent coordination layer (Pillar 3) for delegation accountability. This architectural integration across all three pillars makes the tool orchestration layer a genuine research and engineering contribution rather than a function-calling plugin.\n\nAddressing key methodological challenges. (1) Plan degradation over long task horizons: mitigated by formally specified verification checkpoints and plan monitoring with recovery from known-good intermediate states (T2-03), preventing catastrophic failure propagation. (2) Context window limitations preventing long-horizon coherence: mitigated by the adaptive memory controller deciding what to commit, retrieve, and forget across working, episodic, and semantic stores (T3-02, T3-03). (3) Tool output errors or hallucinated values incorporated into agent state: mitigated by post-invocation verification against expected schemas and domain constraints before any tool result enters agent reasoning. (4) Multi-agent coordination without convergence guarantees: mitigated by mathematically proven delegation and negotiation protocols (T4-01, T4-02), providing formal accountability and convergence assurances not available in emergent LLM-based agent communication.\n\nBuilding on prior and concurrent R&I. Each consortium partner grounds their MAESTRO contribution in documented prior work: ATU's H2020 project on trustworthy AI (WP lead for hybrid AI integration) and contribution to the AI-on-demand platform; BIIS's Horizon Europe project on unified transformers for text and retrieval; CERIA's ERC Advanced Grant on formal methods for multi-agent systems and H2020 project on autonomous multi-agent exploration; NIHS's Horizon Europe participation in the European Health Data Space pilot and H2020 project on AI for clinical imaging diagnostics; ISIA's Horizon Europe project on AI for manufacturing process optimisation and H2020 project on big data for smart factory environments; ELI's H2020 logistics platform project for European supply chains; BAL's Horizon Europe European large language model project and EuroHPC access for large-scale LLM training; and FIIT's H2020 contribution to the AI-on-demand platform and regional EDIH operations.\n\nInterdisciplinary approach. MAESTRO is inherently interdisciplinary, bringing together complementary expertise from six disciplinary communities: (a) Computer science — AI, automated planning, multi-agent systems, and large language models (ATU, CERIA, BIIS, BAL); (b) Formal methods and mathematics — formal protocol specification, convergence proofs, and verification of multi-agent systems (CERIA, ERC Advanced Grant grounding); (c) Medical informatics — clinical decision support systems, HL7 FHIR health data integration, medical NLP, and ethics-approved clinical data management (NIHS); (d) Manufacturing engineering — Industry 4.0 systems, digital twin modelling and simulation, OPC-UA industrial integration, and predictive process optimisation (ISIA); (e) Logistics and supply chain science — multi-actor autonomous planning, disruption recovery, and operational validation across international corridors (ELI); and (f) AI evaluation science and technology transfer — benchmarking methodology, responsible AI assessment, and EDIH-mediated dissemination (FIIT). These disciplines contribute mutually reinforcing methods: formal methods constrain and verify the AI architecture; sector domain expertise drives co-designed validation environments with real operational data; and evaluation science (WP8, FIIT lead) ensures reproducible benchmarking across all demonstrators.\n\nSocial sciences and humanities. MAESTRO is primarily a technical research project developing AI agent architectures, planning engines, memory systems, and coordination protocols. SSH disciplinary contributions are not a principal project scope element. Ethics and accountability considerations — directly relevant to responsible AI development — are addressed through WP1 Task T1-03 (ethics compliance oversight and coordination with the GenAI4EU Hub) and through the auditability requirements embedded in all three research pillars: WP4's structured delegation traces and the External Tool and API Orchestration Layer's full tool-invocation provenance tracking directly support explainability and human oversight of AI decision-making, embedding responsible AI governance proportionate to the project's technical scope without requiring dedicated SSH disciplinary involvement.\n\nGender dimension in research and innovation content.
